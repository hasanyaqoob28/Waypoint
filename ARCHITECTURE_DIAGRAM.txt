│                                                                              │

                                    ↓

LAYER 2: API ENDPOINTS (Next.js Serverless)
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Next.js API Routes                                     │
│                                                                              │
│  POST /api/parse                                                           │
│  • Receives raw booking text from frontend                                │
│  • Forwards text to Google Gemini AI                                      │
│  • Returns parsed structured data                                         │
│                                                                              │
│  GET /api/trips                                                            │
│  • Retrieves all user trips from database                                 │
│  • Returns trip data with all events                                      │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
                    (Sends messy text to AI for parsing)
                                    ↓

LAYER 3: AI PROCESSING (Google Gemini - Vercel AI SDK)
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Google Gemini AI Parser                                  │
│                                                                              │
│  Extracts from messy booking email:                                        │
│  • Flight Number, Airline, Departure Time, Arrival Time                   │
│  • Terminal Number, Gate Number, Aircraft Type                            │
│  • Baggage Carousel Number                                                │
│  • Hotel Name, Hotel Address, Check-in Time, Check-out Time               │
│  • Activity Name, Time, Location                                          │
│  • Transit Information (Uber, Train, etc)                                 │
│                                                                              │
│  Returns: Structured JSON with all extracted data                         │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
                    (Sends parsed data to database)
                                    ↓

COMPLETE DATA FLOW:
1. User pastes booking email → Travelway UI input form
2. Frontend sends raw text → POST /api/parse endpoint
3. API forwards text → Google Gemini AI
4. AI extracts structured data → Returns parsed JSON
5. API stores parsed data → Aurora PostgreSQL database
6. Frontend fetches trips → GET /api/trips endpoint
7. Database returns user's trips with all events
8. React displays → Event timeline, guidance panels, weather
9. User selects event → UI shows context-specific guidance
10. Weather API → Displays destination weather conditions

TECHNOLOGY STACK:
• Frontend: React, Next.js (Vercel deployment)
• API: Next.js serverless functions
• Database: Amazon Aurora PostgreSQL (Primary backend)
• AI: Google Gemini (via Vercel AI SDK)
• External: Open-Meteo Weather API
• Hosting: Vercel (Frontend + API)
• Auth: Session-based (stored in PostgreSQL)
